ObjectLayout implementation


ObjectLayout is a project aiming to provide better control over layout of Java objects in Java heap. For details please refer to www.objectlayout.org


The ObjectLayout library exists in 2 forms: a vanilla Java implementation and an intrinsified implementation. Both implementations provide the same API for the end user of the library.


The vanilla implementation is designed to work with any JDK8+ from any vendor. It is not expected to observe any performance improvements compared to regular java arrays. The purpose of the library is to demonstrate the API completeness and insure smooth code work on any JDK. The source code for the library can be found here: https://github.com/ObjectLayout/ObjectLayout


The intrinsified implementation is designed to work with the modified JDK. The intrinsified implementation uses numerous calls to sun.misc.Unsafe, some of those already exist in JDK8 and others were added in the course of this prototype implementation. This intrinsified implementation is a downstream version of the vanilla code, it is expected that over time both libraries are kept in sync. The source code for intrinsified library is located at https://bitbucket.org/ipk/objectlayoutlibrary


The source code for the modified JDK8 update N class libraries can be found at https://bitbucket.org/ipk/structuredarray-jdk. The source code for the modified hotspot sources can be found at https://bitbucket.org/ipk/structuredarray-hotspot


In the OpenJDK community the most popular way to exchange suggested patches and fixes is though sharing webrevs. Webrev is a set of static html files containing cdiffs, udiffs, wdiffs, sdiffs for all modified files.


The webrev for JDK class libraries changes is located at https://github.com/ivankrylov/OLImplementation

The webrev for hotspot changes is located at https://github.com/ivankrylov/OLImplementation
