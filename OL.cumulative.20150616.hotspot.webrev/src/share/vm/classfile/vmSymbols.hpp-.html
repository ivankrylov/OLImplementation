 597 // Here are all the intrinsics known to the runtime and the CI.
 598 // Each intrinsic consists of a public enum name (like _hashCode),
 599 // followed by a specification of its klass, name, and signature:
 600 //    template(<id>,  <klass>,  <name>, <sig>, <FCODE>)
 601 //
 602 // If you add an intrinsic here, you must also define its name
 603 // and signature as members of the VM symbols.  The VM symbols for
 604 // the intrinsic name and signature may be defined above.
 605 //
 606 // Because the VM_SYMBOLS_DO macro makes reference to VM_INTRINSICS_DO,
 607 // you can also define an intrinsic's name and/or signature locally to the
 608 // intrinsic, if this makes sense.  (It often does make sense.)
 609 //
 610 // For example:
 611 //    do_intrinsic(_foo,  java_lang_Object,  foo_name, foo_signature, F_xx)
 612 //     do_name(     foo_name, "foo")
 613 //     do_signature(foo_signature, "()F")
 614 // klass      = vmSymbols::java_lang_Object()
 615 // name       = vmSymbols::foo_name()
 616 // signature  = vmSymbols::foo_signature()
 617 //
 618 // The name and/or signature might be a "well known" symbol
 619 // like "equal" or "()I", in which case there will be no local
 620 // re-definition of the symbol.
 621 //
 622 // The do_class, do_name, and do_signature calls are all used for the
 623 // same purpose:  Define yet another VM symbol.  They could all be merged
 624 // into a common 'do_symbol' call, but it seems useful to record our
 625 // intentions here about kinds of symbols (class vs. name vs. signature).
 626 //
 627 // The F_xx is one of the Flags enum; see below.
 628 //
 629 // for Emacs: (let ((c-backslash-column 120) (c-backslash-max-column 120)) (c-backslash-region (point) (point-max) nil t))
